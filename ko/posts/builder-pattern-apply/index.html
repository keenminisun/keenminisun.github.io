왜 빌더 패턴이 필요했나

기존 객체 생성 방식은 생성자 인자가 늘어날수록 읽기와 유지보수가 동시에 어려워졌다.

특히 선택 인자가 많아질수록 호출부만 보고 의도를 파악하기 어려웠고, 잘못된 순서로 값을 전달해도 컴파일 단계에서 충분히 드러나지 않았다.

무엇을 바꾸려 했나

내가 해결하고 싶었던 문제는 단순히 “패턴 적용”이 아니라 다음 두 가지였다.

  1. 객체 생성 의도를 코드에서 바로 읽을 수 있게 만들기
  2. 필수 값과 선택 값을 구분해서 실수를 줄이기

기대하는 효과

빌더 패턴은 생성 로직을 분리해 호출부의 가독성을 높여 주고, 나중에 필드가 늘어나도 생성자 시그니처를 무한정 흔들지 않게 해 준다.
